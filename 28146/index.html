4. 部署到 GitHub Pages

最后一步就是把文章发布到线上。直接执行：

（简写命令为 hexo d）

等待几秒，部署完成后，你就可以在自己的 GitHub Pages
地址中访问这篇文章了。
比如：https://你的GitHub用户名.github.io/

5. 小结

这就是用 Hexo 写第一篇博客并发布到 GitHub Pages 的完整流程。
总结一下：

  1. hexo new post "文章名" → 新建文章\
  2. 编辑文章内容\
  3. hexo s → 本地预览\
  4. hexo g → 生成静态文件\
  5. hexo d → 部署到 GitHub Pages
  6. 每次发布文章前记得先用 hexo clean 来清理旧的缓存文件！！！
